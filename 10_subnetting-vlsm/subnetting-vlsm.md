# 10. Subnetting / VLSM

## Subnetting

하나의 네트워크를 여러 개의 서브넷으로 분할하는 기법입니다.
필요한 네트워크 수만큼 Host 비트를 빌려 Network 비트로 사용합니다.

```
※ 서브넷 경계로 자주 쓰이는 마스크 값: 128, 192, 224, 240, 248, 252, 254, 255
```

### EX) 192.168.1.0/24 를 사용하여 각 구간에 IP 할당

```
                  192.168.1.96/27                    192.168.1.128/27
                       Host 2                              Host 2
          R1 ───────────────────── R2 ───────────────────── R3
          │                        │                        │
       Host 30                  Host 30                  Host 30
   192.168.1.0/27           192.168.1.32/27          192.168.1.64/27
```

Network = 5, Host = 30 → `2^n - 2 = 30`, n = 5 (Host 비트 5개) → /27

```
192.168.1.000 00000 = 192.168.1.0/27     (192.168.1.0   ~ 192.168.1.31)
192.168.1.001 00000 = 192.168.1.32/27    (192.168.1.32  ~ 192.168.1.63)
192.168.1.010 00000 = 192.168.1.64/27    (192.168.1.64  ~ 192.168.1.95)
192.168.1.011 00000 = 192.168.1.96/27    (192.168.1.96  ~ 192.168.1.127)
192.168.1.100 00000 = 192.168.1.128/27   (192.168.1.128 ~ 192.168.1.159)
192.168.1.101 00000 = 192.168.1.160/27   (192.168.1.160 ~ 192.168.1.191)
192.168.1.110 00000 = 192.168.1.192/27   (192.168.1.192 ~ 192.168.1.223)
192.168.1.111 00000 = 192.168.1.224/27   (192.168.1.224 ~ 192.168.1.255)
```

## VLSM [Variable Length Subnet Mask]

하나의 네트워크를 효율적으로 사용하기 위해, 가변적으로 서브넷을 분할하여 사용하는 기법입니다.
구간별 필요한 호스트 수에 딱 맞게 마스크 길이를 다르게 적용합니다.

### EX) 192.168.1.0/24 에서 LAN(Host 30)과 라우터 간 링크(Host 2) 할당

```
                 192.168.1.96/30                  192.168.1.100/30
                      Host 2                            Host 2
          R1 ──────────────────── R2 ──────────────────── R3
          │                       │                       │
       Host 30                 Host 30                 Host 30
   192.168.1.0/27          192.168.1.32/27         192.168.1.64/27
```

LAN 구간(Host 30 → /27)은 위 Subnetting 결과를 그대로 사용하고,
남는 `192.168.1.96/27` 블록을 라우터 간 링크용(Host 2 → /30)으로 다시 분할합니다.

```
# 라우터 간 링크 (Host 2 = 2^n - 2, n = 2 → /30)
192.168.1.011 000 00 = 192.168.1.96/30    (192.168.1.96  ~ 192.168.1.99)
192.168.1.011 001 00 = 192.168.1.100/30   (192.168.1.100 ~ 192.168.1.103)
192.168.1.011 010 00 = 192.168.1.104/30   (192.168.1.104 ~ 192.168.1.107)
192.168.1.011 011 00 = 192.168.1.108/30   (192.168.1.108 ~ 192.168.1.111)
192.168.1.011 100 00 = 192.168.1.112/30   (192.168.1.112 ~ 192.168.1.115)
192.168.1.011 101 00 = 192.168.1.116/30   (192.168.1.116 ~ 192.168.1.119)
192.168.1.011 110 00 = 192.168.1.120/30   (192.168.1.120 ~ 192.168.1.123)
192.168.1.011 111 00 = 192.168.1.124/30   (192.168.1.124 ~ 192.168.1.127)
```

## VLSM 종합 예제 ① — 192.168.1.0/24 (Host 50/30/20/10 + 링크)

```
            Host 2          Host 2          Host 2
   R1 ───────────── R2 ───────────── R3 ───────────── R4
   │                │                │                │
Host 50          Host 30          Host 20          Host 10
```

### 1단계: 가장 큰 LAN부터 분할 (Host 50 → /26)

```
192.168.1.00 000000 = 192.168.1.0/26     (192.168.1.0   ~ 192.168.1.63)   ← Host 50
192.168.1.01 000000 = 192.168.1.64/26    (192.168.1.64  ~ 192.168.1.127)
192.168.1.10 000000 = 192.168.1.128/26   (192.168.1.128 ~ 192.168.1.191)
192.168.1.11 000000 = 192.168.1.192/26   (192.168.1.192 ~ 192.168.1.255)
```

### 2단계: 64/26 블록을 Host 30 → /27 로 분할

```
192.168.1.01 0 00000 = 192.168.1.64/27   (192.168.1.64 ~ 192.168.1.95)   ← Host 30
192.168.1.01 1 00000 = 192.168.1.96/27   (192.168.1.96 ~ 192.168.1.127)
```

### 3단계: 128/26 블록을 Host 20 → /27, Host 10 → /28 로 분할

```
# Host 20 (2^n-2≥20, n=5 → /27)
192.168.1.10 0 00000 = 192.168.1.128/27  (192.168.1.128 ~ 192.168.1.159) ← Host 20

# Host 10 (2^n-2≥10, n=4 → /28)
192.168.1.101 0 0000 = 192.168.1.160/28  (192.168.1.160 ~ 192.168.1.175) ← Host 10
```

### 4단계: 라우터 간 링크 Host 2 → /30

```
192.168.1.1011 00 00 = 192.168.1.176/30
192.168.1.1011 01 00 = 192.168.1.180/30
192.168.1.1011 10 00 = 192.168.1.184/30
```

## VLSM 종합 예제 ② — 211.241.20.0/24 (Host 60/80/25/12 + 링크)

```
            Host 2          Host 2          Host 2
   R1 ───────────── R2 ───────────── R3 ───────────── R4
   │                │                │                │
Host 60          Host 80          Host 25          Host 12
```

가장 큰 호스트부터 순서대로 할당합니다.

```
# Host 80 (n=7 → /25)
211.241.20.0 0000000 = 211.241.20.0/25     (211.241.20.0   ~ 211.241.20.127)  ← Host 80

# Host 60 (n=6 → /26)  : 128/25 블록 분할
211.241.20.1 0 000000 = 211.241.20.128/26  (211.241.20.128 ~ 211.241.20.191)  ← Host 60

# Host 25 (n=5 → /27)  : 192/26 블록 분할
211.241.20.11 0 00000 = 211.241.20.192/27  (211.241.20.192 ~ 211.241.20.223)  ← Host 25

# Host 12 (n=4 → /28)  : 224/27 블록 분할
211.241.20.111 0 0000 = 211.241.20.224/28  (211.241.20.224 ~ 211.241.20.239)  ← Host 12

# 라우터 간 링크 Host 2 (n=2 → /30) : 240/28 블록 분할
211.241.20.1111 00 00 = 211.241.20.240/30
211.241.20.1111 01 00 = 211.241.20.244/30
211.241.20.1111 10 00 = 211.241.20.248/30
211.241.20.1111 11 00 = 211.241.20.252/30
```

## VLSM 대규모 예제 ③ — 133.200.0.0/16 (Host 4000 ~ 2)

```
요구 Host: 4000, 3500, 2500, 1000, 500, 230, 25, 22, 2~7

133.200.0000 0000.00000000 = Host 4000 = 133.200.0.0/20   (133.200.0.0  ~ 133.200.15.255)
133.200.0001 0000.00000000 = Host 3500 = 133.200.16.0/20  (133.200.16.0 ~ 133.200.31.255)
133.200.0010 0000.00000000 = Host 2500 = 133.200.32.0/20  (133.200.32.0 ~ 133.200.47.255)
133.200.0011 00 00.00000000 = Host 1000 = 133.200.48.0/22 (133.200.48.0 ~ 133.200.51.255)
133.200.0011 01 0 0.00000000 = Host 500 = 133.200.52.0/23 (133.200.52.0 ~ 133.200.53.255)
133.200.0011 01 1 0.00000000 = Host 230 = 133.200.54.0/24 (133.200.54.0 ~ 133.200.54.255)
133.200.0011 01 1 1.000 00000 = Host 25 = 133.200.55.0/27
133.200.0011 01 1 1.001 00000 = Host 22 = 133.200.55.32/27

# 라우터 간 링크 (Host 2 → /30)
133.200.0011 01 1 1.010 000 00 = 133.200.55.64/30
133.200.0011 01 1 1.010 001 00 = 133.200.55.68/30
133.200.0011 01 1 1.010 010 00 = 133.200.55.72/30
133.200.0011 01 1 1.010 011 00 = 133.200.55.76/30
133.200.0011 01 1 1.010 100 00 = 133.200.55.80/30
133.200.0011 01 1 1.010 101 00 = 133.200.55.84/30
133.200.0011 01 1 1.010 110 00 = 133.200.55.88/30
133.200.0011 01 1 1.010 111 00 = 133.200.55.92/30
```

> 💡 VLSM 핵심: **호스트 수가 큰 네트워크부터** 순서대로 할당해야 주소 낭비 없이 분할됩니다.
