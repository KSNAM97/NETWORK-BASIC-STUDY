# 11. IP Address Summary (집약)

IP 주소를 다시 최적화하기 위해, 여러 개의 작은 네트워크를
하나의 큰 네트워크로 집약(Summary)하여 사용하는 기법입니다.
(Subnetting/VLSM의 반대 개념)

## IP 주소 집약의 장점

1. **Router의 Routing Table을 최소화**할 수 있다.
   - Subnetting/VLSM으로 분할하면 서로 다른 네트워크가 되어 라우팅 테이블에 각각 등록된다.
   - 집약하면 등록 수가 줄어 Memory 소모와 Lookup Delay를 줄일 수 있다.
2. **Flapping(경로 흔들림)으로 인한 영향을 IP 집약으로 차단**할 수 있다.

---

## 집약 계산 예제

### EX1) 172.168.0.0/24 ~ 172.168.3.0/24 집약

```
# 공통 비트 찾기 (세 번째 옥텟)
172.168.000000 00.00000000
172.168.000000 01.00000000
172.168.000000 10.00000000
172.168.000000 11.11111111
-----------------------------
# Wildcard Mask
0.0.000000 11.11111111  = 172.168.0.0  0.0.3.255

# CIDR (SubnetMask)
255.255.111111 00.00000000 = 172.168.0.0  255.255.252.0 = 172.168.0.0/22
```

### EX2) 172.168.0.0/24 ~ 172.168.7.0/24 집약

```
172.168.00000 000.00000000
172.168.00000 001.00000000
       ~
172.168.00000 111.11111111
-----------------------------
# Wildcard Mask
0.0.00000 111.11111111  = 172.168.0.0  0.0.7.255

# CIDR
255.255.11111 000.00000000 = 172.168.0.0  255.255.248.0 = 172.168.0.0/21
```

### EX3) 172.168.4.0/24 ~ 172.168.7.0/24 집약

```
172.168.000001 00
172.168.000001 01
172.168.000001 10
172.168.000001 11
-----------------------------
255.255.111111 00 = 172.168.4.0  255.255.252.0 = 172.168.4.0/22
```

### EX4) 172.168.8.0/24 ~ 172.168.15.0/24 집약

```
172.168.00001 000
       ~
172.168.00001 111
-----------------------------
255.255.11111 000 = 172.168.8.0  255.255.248.0 = 172.168.8.0/21
```

### EX5) 172.168.0.0/24 ~ 172.168.10.0/24 집약

```
172.168.0000 0000
172.168.0000 0001
       ~
172.168.0000 1111
-----------------------------
255.255.1111 0000 = 172.168.0.0  255.255.240.0 = 172.168.0.0/20
```

> ※ 11까지만 필요하지만 비트 경계상 0~15(/20)로 묶입니다.

## Classful 주소 집약 (Auto-Summary)

Subnetting/VLSM으로 분할한 네트워크를 다시 Class의 기본 SubnetMask로 집약하는 방식입니다.

```
                 Serial 1/0                 Serial 1/1
         R1 ─────────────────────────────── R2
  C 172.16.0.0/24 Fa0/0.10      X 172.16.0.0/16 Serial 1/1  ← B class 기본 마스크로 집약 (255.255.0.0)
  C 172.16.1.0/24 Fa0/0.20
  C 172.16.2.0/24 Fa0/0.30
  C 172.16.3.0/24 Fa0/0.40
```

> ⚠️ Classful 집약은 주소 손실 범위가 크기 때문에 Loop를 발생시킬 수 있고,
> Class 경계가 그대로 적용되지 않는 환경에서는 문제가 생길 수 있어 권장되지 않습니다.
