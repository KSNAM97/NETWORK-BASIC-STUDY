# 12. Wildcard Mask

- EIGRP, OSPF Routing Protocol에서 네트워크 범위를 지정할 때 사용된다.
- ACL 같은 Filtering 기능에서, Filtering을 실시할 네트워크 범위를 지정할 때 사용된다.

---

## SubnetMask vs WildcardMask 비교

### SubnetMask

32bit로 구성되며 1과 0이 **연속적인** 형태만 가집니다.

```
1 = Network (서브넷)
0 = Host    (논 서브넷)
```

### WildcardMask

32bit로 구성되며 1과 0이 **불연속적인** 형태도 가집니다.
SubnetMask와 의미가 **반대**입니다.

```
0 = 체크 (서브넷 / 일치해야 함)
1 = 무시 (논 서브넷 / 아무 값이나 가능)
```

---

## 변환 예시

```
192.168.1.0/24
   → 192.168.1.0   0.0.0.255    (192.168.1.0  ~ 192.168.1.255)

172.16.0.0/16
   → 172.16.0.0    0.0.255.255  (172.16.0.0   ~ 172.16.255.255)

10.0.0.0/8
   → 10.0.0.0      0.255.255.255 (10.0.0.0   ~ 10.255.255.255)

192.168.1.0/27
   → 192.168.1.0   0.0.0.31     (192.168.1.0  ~ 192.168.1.31)

192.168.1.64/26
   → 192.168.1.64  0.0.0.63     (192.168.1.64 ~ 192.168.1.127)

192.168.1.1     0.0.0.0  = Host (단일 호스트 지정)
```

> 💡 `/24` → 255.255.255.0 (SubnetMask) → 0.0.0.255 (WildcardMask)
> 각 옥텟을 `255 - SubnetMask값` 으로 빠르게 계산할 수 있습니다.

## 범위 지정 계산 예제

### EX1) 172.168.0.0/24 ~ 172.168.3.0/24

```
172.168.000000 00.00000000
172.168.000000 01.00000000
172.168.000000 10.00000000
172.168.000000 11.11111111
-----------------------------
# WildcardMask
0.0.000000 11.11111111 = 172.168.0.0  0.0.3.255

# CIDR
255.255.111111 00.00000000 = 172.168.0.0  255.255.252.0
```

### EX2) 172.168.0.0/24 ~ 172.168.7.0/24
